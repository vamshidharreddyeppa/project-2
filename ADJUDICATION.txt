Final verified verdict:
My final verdict is that this was a memory sizing failure. The Ollama process loaded deepseek-7b FP16 with a large KV cache, and the machine did not have enough memory for that workload. The kernel killed the Ollama process with the OOM killer. The real fix is to use a smaller or quantized model, reduce context and batch size, or upgrade the machine memory. Restarting model-server.service is only a temporary symptom patch because the same model can be killed again.
